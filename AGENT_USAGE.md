# AGENT_USAGE.md

## Objetivo
Este repositorio está preparado para trabajar con agentes (Codex, Copilot, etc.)
para reducir carga manual en documentación.

---

## Cómo usar el agente en este proyecto

### 1. NO empezar editando
Primero pedir análisis:

Ejemplo:
"Revisa este documento contra la rúbrica. No edites aún."

---

### 2. Luego hacer cambios dirigidos
Ejemplo:
"Corrige solo formato markdown en README.md. No cambies contenido."

---

### 3. Evitar prompts vagos
❌ "arregla todo"
❌ "mejora el documento"

✔ Usar instrucciones específicas:
- qué parte del archivo
- qué tipo de cambio
- qué NO cambiar

---

## Flujo recomendado

1. Diagnóstico
2. Validación con el equipo
3. Edición puntual con agent
4. Revisión final

---

## Buenas prácticas

- Cambios pequeños y revisables
- No reescribir todo el documento
- Validar contra la rúbrica siempre
- Revisar diffs antes de aceptar

---

## Ejemplos útiles

### Revisar estructura
"Analiza README.md y dime qué secciones están incompletas."

### Validar rúbrica
"Contrasta este documento con los criterios del avance 1."

### Formato
"Corrige headers y listas markdown sin cambiar contenido."

---

## Qué NO hacer

- No aceptar cambios masivos sin revisar
- No confiar en contenido inventado
- No usar el agente como reemplazo de criterio académico

## NOTA:
El contenido de AGENTS.md es simple y provisional, agregaremos mayor detalle para que se puedan validar correctamente las entregas del reporte.
